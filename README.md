# When Depth Adds Nothing: Probing Recurrent-Depth Transformers

This repository contains the code, data-generation pipeline, and paper source for *When Depth Adds Nothing: A Negative Result on Probing Recurrent-Depth Transformers*, accepted as a poster at the **MusIML Workshop @ ICML 2026** and under review at **ACL ARR**.

The paper asks whether the depth dimension of a recurrent-depth transformer (RDT) provides a useful monitoring affordance for detecting reward-hacking behavior, given that chain-of-thought monitoring is structurally inapplicable to latent-reasoning architectures. The short answer is **no, not in the way one might first expect** — and the experiments are designed to characterize *why* with the controls necessary to interpret the result.

---

## Headline findings

The paper runs three experiments on a small (~2.3M parameter) recurrent-depth transformer trained from scratch.

| Experiment | What it tests | Headline number |
|---|---|---|
| **exp1** — depth-indexed probes | RL-induced hacking signature across loop depths | Task probe AUROC = 1.0 at every depth |
| **exp1b** — input-layer baselines | Whether the loop adds anything over the input | 1-bit "leak present" feature reaches AUROC = 0.99 |
| **exp4** — steganographic exploit | Whether a harder, non-linear exploit changes the conclusion | Depth-1 already at AUROC = 1.0; depth-8 identical |

The cleanest single-sentence summary: **for every exploit class we tested, the recurrent loop contributes no monitoring information that simpler controls (input embeddings, or depth-1 representations) cannot recover.** This is a negative result with a constructive methodological consequence: depth-probing studies on RDTs should run both input-layer and depth-1 baselines before claiming the loop is the site of relevant computation.

We do **not** claim that depth-indexed probing is useless in general. Section 6.3 of the paper identifies exploit classes (generation-time hacks, multi-step reasoning hacks) for which our experiments would not predict a negative result, and we frame them as direct hypotheses for follow-up.

## Requirements

- **OS:** Linux or macOS. Tested on Ubuntu 22.04.
- **GPU:** Single NVIDIA A10 (24GB VRAM) or equivalent. CPU-only runs will work but are slow.
- **Python:** 3.10–3.12.
- **CUDA:** 12.1 or compatible.
- **Disk:** ~500MB for checkpoints, negligible for code and results.

Install dependencies with pinned versions:

```bash
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

The full dependency list lives in `requirements.txt`; the load-bearing pins are `torch>=2.5,<2.12`, `transformers>=4.46,<5.0`, `scikit-learn>=1.5,<1.8`, and `scipy>=1.13`.

## Reproduction

The full pipeline runs in approximately **45 minutes wall-clock** on a single A10. Each step writes its outputs to `results/<exp_name>/` and produces a human-readable `summary.txt` alongside the structured JSON and CSV.

### Step 1 — Train the baseline RL-hacked model (~20 min)

```bash
python src/common.py \
    --train-baseline \
    --out-dir checkpoints/baseline \
    --pretrain-steps 1300 \
    --rl-steps 1300 \
    --checkpoint-every 90 \
    --seed 0
```

The reward curve should climb from ~0.5 (random) to ~0.99 (model has adopted the leak-copying strategy).

### Step 2 — Run the depth-indexed probe experiment (~2 min)

```bash
python src/exp1_probes.py \
    --base-ckpt checkpoints/baseline/pretrained.pt \
    --rl-ckpt   checkpoints/baseline/rl_step_01300.pt \
    --out-dir   results/exp1_probes \
    --n-probe 4000 --n-eval 2000 \
    --seed 42
```

Expected verdict: `NULL_RESULT` — no hacking-prone depths identified by the pre-registered criteria.

### Step 3 — Run the input-layer baseline (~1 min)

```bash
python src/exp1b_input_probe.py \
    --base-ckpt    checkpoints/baseline/pretrained.pt \
    --rl-ckpt      checkpoints/baseline/rl_step_01300.pt \
    --exp1-results results/exp1_probes/probe_results.json \
    --out-dir      results/exp1b_input_probe \
    --seed 42
```

Expected verdict: `DEPTH_REDUNDANT` — the loop contributes nothing over input-layer probing. The headline number is that a 1-bit "leak-present" input feature alone achieves AUROC = 0.99.

### Step 4 — Steganographic-exploit follow-up (~20 min)

```bash
# Train a fresh model on the harder task
python src/exp4_steganographic.py train-baseline \
    --out-dir checkpoints/stego_baseline \
    --pretrain-steps 1300 \
    --rl-steps 1300 \
    --checkpoint-every 90 \
    --seed 0

# Probe it
python src/exp4_steganographic.py probe \
    --base-ckpt checkpoints/stego_baseline/pretrained.pt \
    --rl-ckpt   checkpoints/stego_baseline/rl_step_01300.pt \
    --out-dir   results/exp4_steganographic \
    --seed 42
```

Expected verdict: `WEAK_SUCCESS_DEPTH_ADDS_VALUE_NO_LOCALITY` — depth probes outperform mean-pooled input embeddings as predicted, but depth-1 already reaches AUROC = 1.0 and deeper iterations add nothing. The loop is still functionally invisible; the gap to the input baseline is created by the prelude blocks (which run once, before the loop), not by loop iteration.

### Step 5 — Generate the main figure

```bash
python src/make_main_figure.py \
    --exp1   results/exp1_probes/probe_results.json \
    --exp1b  results/exp1b_input_probe/input_probe_results.json \
    --out    paper/figures/fig1_auroc_vs_depth.pdf
```

### Step 6 — Compile the paper

```bash
cd paper/
pdflatex main.tex
bibtex main
pdflatex main.tex
pdflatex main.tex
```

## Summary of all three results

The numbers below are reproduced from the result JSONs in `results/`. Each represents a single run with the seeds documented in the reproduction commands above.

### Experiment 1 — Depth-indexed probes (input-channel exploit)

| Depth | Task AUROC | Control AUROC | Base AUROC | Gap (task − base) | p (BH-corrected) |
|------:|-----------:|--------------:|-----------:|-----------------:|----------------:|
| 1 | 1.000 | 0.470 | 1.000 | +0.000 | 0.899 |
| 2 | 1.000 | 0.470 | 1.000 | +0.000 | 0.899 |
| 3 | 1.000 | 0.469 | 1.000 | +0.000 | 0.899 |
| 4 | 1.000 | 0.469 | 1.000 | +0.000 | 0.899 |
| 5 | 1.000 | 0.468 | 1.000 | +0.000 | 0.899 |
| 6 | 1.000 | 0.468 | 1.000 | +0.000 | 0.899 |
| 7 | 1.000 | 0.467 | 1.000 | +0.000 | 0.899 |
| 8 | 1.000 | 0.467 | 1.000 | +0.000 | 0.899 |

**Verdict:** `NULL_RESULT`. Task probes achieve perfect discrimination but base probes do too — RL did not install the signature, and there is no depth localization. Eval n = 1,981 (1,928 hacking / 2,038 genuine labelled examples in the probe training set).

### Experiment 1b — Input-layer baselines

| Feature set | #features | AUROC [95% BCa CI] |
|---|---:|---|
| `leak_token_only` | 1 | 0.9897 [0.9841, 0.9937] |
| `bag_of_tokens` | 64 | 1.0000 [1.0000, 1.0000] |
| `input_embed_base` | 192 | 1.0000 [1.0000, 1.0000] |
| `input_embed_rl` | 192 | 1.0000 [1.0000, 1.0000] |

**Verdict:** `DEPTH_REDUNDANT`. A single bit of input information closes the gap; the recurrent loop adds no monitoring information beyond what is available from the input tokens.

### Experiment 4 — Steganographic exploit

The gold answer is `(x_pos_a + x_pos_b) mod 10` for fixed positions `pos_a, pos_b` in a 6-digit input. Linear input probes on mean-pooled embeddings cannot recover this (averaging destroys positional information).

| Probe | AUROC [95% BCa CI] |
|---|---|
| Mean-pooled input embeddings (RL model) | 0.494 [0.469, 0.519] |
| Depth-1 task probe | 1.000 |
| Depth-8 task probe | 1.000 |
| Depth-1 base (pre-RL) probe | 1.000 |

**Verdict:** `WEAK_SUCCESS_DEPTH_ADDS_VALUE_NO_LOCALITY`. The gap between depth probing and mean-pooled input baseline is real and large (+0.506, p_BH = 0.0005 at every depth). But the gap is created by the prelude blocks (which apply attention once before the loop), not by loop iteration: depth 1 is already at ceiling and depths 2 through 8 are identical. The loop remains functionally invisible to monitoring.

The methodological lesson from exp4 sharpens the lesson from exp1+exp1b: depth-probing studies on RDTs should report both an input-layer baseline (to control against trivial input-determined features) and a depth-1 baseline (to control against prelude contribution). If depth-T probing matches both baselines, the loop is doing no work that probing can detect.

## Limitations to flag explicitly

This is a small-scale study with deliberate scope restrictions:

- **Scale:** 2.3M-parameter model trained from scratch. Results may not transfer to billion-scale RDTs like Huginn-3.5B without further work.
- **Exploit class:** two exploit classes (direct input-channel leak and 2-position steganographic). Generation-time exploits, multi-step reasoning hacks, and truly adversarial exploits remain untested.
- **Architecture:** one variant of the input-injection RDT family. Universal Transformers, looped transformers without input injection, and continuous-CoT architectures are not tested.
- **Single seed for RL training.** A multi-seed replication is planned for any camera-ready version.
- **Linear probes only.** Nonlinear probes might extract information that linear probes cannot.

Full limitations discussion is in §7 of the paper.
